# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Added

- **`gis_utils.cad` — styled GeoPackage → DXF emitter (one-way)** — new module `cad/` that writes vector layers to a DXF driven by a strict style map, with no `project_settings`/`all_layers` coupling and no sync (Phase-4 R1: rewritten with reference to Python-ACAD-Tools, not migrated). Entry point `export_layers(specs, *, styles, out_dxf, crs, template_dxf=None, dxfversion="R2010")` over `LayerSpec(source, target_layer, style, layer=None, label_field=None)`: reads each source (GeoPackage canonical; any geopandas-readable vector), verifies its CRS against the required project `crs` — a mismatch is a hard error, never a silent reproject (gis-safety) — creates/styles the target DXF layer (colour/linetype/lineweight/transparency/plot/lock/freeze), writes polygon/line/point geometries (optional per-style `HATCH` and centroid `MTEXT` labels), and either starts fresh or adds onto a template DXF whose content is kept (title block, frame, linetypes). Strict style schema `cad_styles.yaml` (`schema_version: 1`, closed snake_case field menus — unknown field is a hard `StyleError`) via `load_styles`; ACI colour table + resolution (`resolve_color`, unknown name is a hard error) salvaged from the reference's `aci_colors.yaml` + `dxf_utils.get_color_code`/`create_hatch`/`update_layer_properties`, decoupled from its `ProjectLoader`. Verified against the real Georgendorf core layers (Geltungsbereich/Baufeld/Baugrenze): layers/entity-counts/colours structurally match the delivered DXF (Baugrenze exact incl. `ACAD_ISO11W100`; `red` → ACI 10 exactly as the reference table + real DXF). Scope of this cut is the emitter core only — legend, viewport, blocks, operations, manifest emission, and workflow-template registration follow in later slices. `tests/test_cad_styles.py` + `tests/test_cad_emit.py` (schema validation, contract errors, emit roundtrip).

- **Niedersachsen source corpus** — seven LGLN-OpenGeoData recipes (kein Passwort, © GeoBasis-DE/LGLN, EPSG:25832), WFS wherever the product exists as WFS: `ni_alkis` (ALKIS vereinfacht `ave:` WFS — Flurstücke, Katasterbezirke, Verwaltungseinheiten, Gebäude, Nutzung; ad-hoc bbox funktioniert, kein StoredQuery nötig — Pendant zu `sh_alkis`/`th_alkis`), `ni_atkis` (Basis-DLM WFS-SF mit Straßen-/Bahn-Klassifizierungs-Cascade wie `mv_atkis`, zusätzlich Gewässer-, Leitungs-, Ortslage- und Wald-Layer), `ni_boris` (Bodenrichtwerte zonal + lagetypisch, WFS), `ni_dop` (DOP20 WMS; historische DOPs via `doph_wms` im Kommentar), `ni_dtk25`/`ni_dtk50`/`ni_dtk100` (DTK WMS, Farbe; Grau-Variante als `ni_dtk*_grau` dokumentiert). Alle Verbindungen gegen die Dienste verifiziert (GetCapabilities geparst, Test-Downloads am Projektgebiet Cuxhaven).
- **`wfs.download` — `connection.bbox_kvp: true`** — opt-in plain-KVP-BBOX GetFeature statt OGR-WFS-Treiber. Nötig für Dienste, bei denen OGRs generierter FES-Filter die Geometrie-Property im falschen Namespace adressiert (LGLN `boris_wfs`: Geometrie ist `adv:position`, OGR filtert auf `boris:position` → Server liefert stillschweigend 0 Features). Der bislang ungenutzte `_wfs_get_feature_url`-Builder sendet BBOX jetzt spec-konform in x,y-Achsordnung (EPSG-projizierte CRS sind E,N definiert).

- **`runner` — per-step `conda_env:`** — a workflow step can override the project-wide conda env, e.g. a headless-PyQGIS layout step (`conda_env: qgis-ltr`) inside a pipeline that otherwise runs in a plain geopandas env.
- **Runner test suite** — `tests/test_runner.py`: 27 unit tests (pure `tmp_path`, no QGIS/network) for the runner core semantics — topological ordering incl. definition-order stability and diamond deps, make-style staleness over `inputs:` / `script:` / recipe `input_boundary:`, and error cases (circular dependency, unknown `depends_on`, missing `input_boundary`). `pytest` added as a dev dependency; run with `uv run pytest`.

### Fixed

- **README doc-drift** — the "Starting a new project" section claimed `gis-workflow init` creates a `CLAUDE.md` (it creates the `PROJECT_DIRS` folders + `workflow.yaml`, no CLAUDE.md), and "Full API reference" linked a non-existent `CLAUDE.md`. Both corrected to the real `init` output and to MCP-`catalog` / `gis-workflow catalog` discovery.
- **`atkis._fetch_features_by_ids` — RESOURCEID-KVP statt XML-ResourceId-Filter** — die niedersachsen.de-WAF vor den LGLN-Diensten weist XML in Query-Parametern komplett zurück (HTML-Fehlerseite statt WFS-Antwort). Der WFS-2.0-Standard-Parameter `resourceID=id1,id2,…` funktioniert bei LGLN **und** geodaten-mv (Regression gegen die `mv_atkis`-Klassifizierungs-Cascade getestet).

- **GeoPackage is the default vector output** — codified in the always-on `gis-safety` skill: write `.gpkg` (`to_file(path, driver="GPKG")`) under `Geodaten/`; Shapefile only when an external recipient requires it (no 10-char field truncation, single-file, multi-layer, UTF-8/NULL). The `workflow-authoring` layout table marks `Geodaten/` as GeoPackage. The library already writes GPKG in nearly all paths; this makes the convention explicit so projects stop emitting `.shp`. Also swept stale `Shape/…​.shp` examples across templates + skills (`buffer_zones`, `apply_qml_style`, `polygon_difference`, `point_buffer_union`, and the `buffer-zones-workflow` / `dxf-lageplan-extraction` / `qgis-mcp-integration` / `schutzgebiete-analyse` skills) to `Geodaten/…​.gpkg` (genuine received `Grundlagen/*.shp` inputs left as-is).
- **Canonical project layout** — `gis_utils.runner.PROJECT_DIRS` defines the standard folder names once (`Grundlagen/`, `Geodaten/`, `Karten/`, `Reports/`, `scripts/`). `gis-workflow init` now scaffolds all five (was `scripts/` only) and the generated `workflow.yaml` example uses `Geodaten/` (was the stray `Shapes/`). The `workflow-authoring` skill documents the layout as mandatory — stops every project re-inventing folder names.
- **`render_layout_template` fills HTML multiframes + re-seeds legend / scale bar** — placeholder fill now also covers `QgsLayoutItemHtml` multiframes (PBS templates put the title there, so the label-only loop left `{{title}}` unfilled and HTML *labels* render async/unreliably); after the map extent/theme is set it re-seeds the legend from the map theme and re-fits the scale bar via `applyDefaultSize` (both previously manual-only gotchas). New `dxf-lageplan-extraction` "3D-Solids (ACIS)" note: vertex projection works for box solids (Gebäude-Sohle), but curved/spline solids (Tank/Zylinder) have no usable vertices and no open-source Python ACIS kernel tessellates them — reconstruct from a dimension label (`solid3d_to_circle`) or explode to 2D in CAD.
- **Layout maps follow a map theme** — `qgis_bridge.define_map_theme(name, visible_layers)` builds/updates a QGIS map theme (idempotent, layers matched by name, builds a valid `QgsLayerTreeModel` so the `createThemeFromCurrentState` call can't segfault QGIS). `render_layout_template(..., map_theme=…)` and the `layout_from_qpt` template's `map.theme:` make the `main_map` item **follow** that theme instead of the live canvas, so canvas edits never silently change an exported layout. Prior `setFollowVisibilityPreset(False)` is now the no-theme fallback with a printed advisory. Convention codified in the `gis-safety` (any-project net), `qgis-mcp-integration`, and `layout-from-template` skills.
- **`runner` — generic `inputs:` step field** — declare the files a step reads (incl. upstream steps' outputs); under `run: auto` the step re-runs when any input is newer than its outputs (make-style staleness) and re-runs cascade downstream in one `gis-workflow run`. Generalizes the recipe-only `input_boundary` mtime check to all steps. Without it `auto` skips whenever outputs merely exist, so an edited input (e.g. a re-digitized shapefile) was silently ignored. A step's own `script:` is now an implicit input too (make-style: target depends on its recipe), so editing the code re-runs the step and cascades downstream. `workflow-authoring` skill documents the field and connects it to the report convention.
- **`reporting.conflict_matrix`** — per source feature × named target: overlap area, else minimum distance. Generalizes `intersection_areas` to several named targets and to the no-overlap (distance) case. Reusable for any proximity/overlap conflict analysis (Baumkronen ↔ Bauteile, Vorhaben ↔ Schutzgebiete, …). Returns a tidy DataFrame (`source, source_area_m2, target, overlap_m2, distance_m, contact`).
- **`workflow-authoring` skill — "Reports are steps too" convention** — a report whose figures derive from the geodata is a pipeline step generated mechanically (static prose + computed values via `conflict_matrix` / `area_report` / `markdown_table`), never hand-typed numbers. Description widened to trigger on report / write-up requests.
- **`recipes/mv_dop.yaml`** — MV DOP20 Open Data WMS recipe (© GeoBasis-DE/M-V). Pair to `sh_dop20`; supports EPSG:25833 natively.
- **`templates/lines_to_polygon`** — generic counterpart to `dxf_lines_to_polygon`; converts (Multi)LineString features from any OGR source (SHP/GPKG/GeoJSON) into a closed polygon.
- **`templates/fetch_flurstuecke`** — thin workflow wrapper around `find_flurstuecke()` so ALKIS parcel lookups can live in `workflow.yaml` as a one-step template.
- **`reporting.area_length_report`** — markdown report combining perimeter (length) and area per layer; auto-detects metrics by geometry type (polygons → length + area; lines → length). Override via `metrics_per_layer`.
- **`recipes.qgis_uri`** — build a QGIS layer URI string (WMS / WFS / XYZ) from a recipe so qgis-mcp's `add_web_layer` (or `QgsRasterLayer`/`QgsVectorLayer` directly) can consume recipes without hand-rolled URIs.

### Changed

- **New-project template no longer defaults to conda** — post uv-migration, `gis-workflow init` generates a `workflow.yaml` without the old `project.conda_env: gis` line, so steps run in the current (uv-managed) Python environment by default. The per-step / project `conda_env:` override still works (see the per-step `conda_env:` note above and `run_step()`); it is now purely opt-in for steps that genuinely need a separate conda env (e.g. PyQGIS). No behaviour change for existing projects that set `conda_env:` explicitly.
- **`templates/polygon_difference`** — `overlay:` now accepts either a single path or a list of paths (unioned before subtraction). `overlay_layer:` correspondingly accepts a list or a single string. Single-string usage is fully back-compat.
- **`recipes/mv_alkis.yaml`** — description now warns against loading the full Flurstück layer as a WFS layer in QGIS and attribute-filtering client-side (60s+ timeouts on ~2M features); points users at `find_flurstuecke()` / `fetch_flurstuecke`.
- **`gis-workflow init`** — default `workflow.yaml` no longer emits an `Example step` referencing a non-existent `scripts/example.py` (which made `gis-workflow run` fail on every fresh project). New template starts with `steps: []` and a commented-out example block.
- **`find_flurstuecke`** error when no search scope is provided now lists all four options (`input_boundary` / `extent` / `gemarkung_schluessel` / `gemeinde_schluessel`) with `input_boundary` marked as recommended.
